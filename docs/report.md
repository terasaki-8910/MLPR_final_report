# 異種書誌LODへの汎用RDF→グラフ変換とR-GCNノード分類によるクラス分離度の横断比較

> 注: 本ドラフトは最終数値を反映する前の骨子である。`<<...>>` で囲んだ箇所は
> `docs/results_*.md`・`docs/class_separation_summary.md` の確定値で差し替える。

## 1. 目的

本研究の目的は、**スキーマ・言語・ドメインが大きく異なる実世界の書誌/典拠 Linked Open Data
(LOD) に対して、単一の RDF→グラフ変換パイプラインを設定ファイルの差し替えだけで適用し、同一の
R-GCN アーキテクチャでノード分類を行い、学習された埋め込み空間のクラス分離度をエンドポイント
横断で比較すること**である。

ここで明確にしておくべき点がある。**採用するモデルである R-GCN (Schlichtkrull et al., 2018)
には新規性が一切ない。** R-GCN は RDF ナレッジグラフのノード分類における最も標準的な手法であり、
原論文自体が AIFB・MUTAG・BGS・AM という RDF ベンチマークで評価している。したがって本研究が
主張しうる新規性は、モデル側ではなく**評価・分析側**に限定される。すなわち、(a) スキーマも言語も
ドメインも異なる複数の実世界 LOD を、エンドポイント固有のコードを書かずに設定ファイルのみで
統一的に扱えること、(b) その上で「クラスの分離しやすさ」をエンドポイント横断で比較する軸を
提示すること、の2点である。この2点は地味だが具体的で検証可能である。

## 2. 関連研究

- **R-GCN (Schlichtkrull et al., 2018, "Modeling Relational Data with Graph Convolutional
  Networks")**: 本研究が採用するモデルそのもの。リレーションタイプごとに重み行列を持つ
  グラフ畳み込みにより、多関係グラフ（RDF）上のノード分類・リンク予測を行う。本研究はこれを
  そのまま用い、モデルの新規性は主張しない。
- **AutoRDF2GML (Färber et al., ISWC 2024)**: RDF から異種グラフ機械学習用データセットを
  自動生成する枠組みで、RDF→グラフ変換という点で本研究に最も近い先行研究である。本研究は
  比較対象としてではなく、「本研究が簡易版として何を諦めたか」を説明する参照点として言及する。
  具体的には、本研究はオントロジーアライメントやエンティティリンク（owl:sameAs 解決）を
  スコープ外とし、各エンドポイントを独立グラフとして扱う点で AutoRDF2GML より大幅に簡略である。

## 3. 方法

### 3.1 対象エンドポイントとスコープ

当初は書誌/典拠系4エンドポイント（DBLP・NDL Web NDL Authorities・ICCU SBN・Cervantes Virtual）
を計画したが、提出期限の制約により、確定作業まで完了していた以下の2件に縮小した（縮小理由は
5節）。

| エンドポイント | 内容 | 分類対象クラス | ラベルプロパティ | クラス（top_k=5） |
|---|---|---|---|---|
| NDL Web NDL Authorities | 日本語 主題・人物典拠 | `skos:Concept` | `skos:inScheme` | personalNames, corporateNames, topicalTerms, geographicNames, uniformTitles |
| DBLP | 計算機科学文献 書誌 | `dblp:Publication` | `dblp:bibtexType` | Article, Inproceedings, Phdthesis, Incollection, Proceedings |

NDL は書誌ノードを持たないため分類対象が典拠ノード自体である点で、書誌レコードを分類する DBLP
とスキーマ概念が根本的に異なる。この異質性を、汎用パイプラインの設定駆動性を示す実例として
積極的に用いる。

対象クラスとラベルプロパティのURIは決め打ちせず、(1) `?s a ?class` のインスタンス数集計による
クラス発見、(2) 述語プロファイル（被覆率・値型・カーディナリティ）、(3) ラベル候補述語の
GROUP BY 全数集計による分布検証、の3段階を経て確定した。特に (3) では、200件サンプルの述語
プロファイルが物理格納順に強く相関した非ランダムな分布を返すこと（DBLP の `bibtexType` で
200件サンプルは Incollection 94%だが真の分布は Article 50.7% / Inproceedings 45.4%）を実証し、
サンプル統計を分布推定に用いてはならないことを確認した。

### 3.2 ラベル抽出（層化抽出）

対象クラスのインスタンスを素朴に `LIMIT max_nodes` で取得すると、上記の物理格納順バイアスが
そのまま抽出結果に混入する。これを避けるため、全数集計で得た top_k ラベル値ごとの真の件数比に
`max_nodes`（=15,000）を按分し、ラベル値ごとに個別クエリで抽出する層化抽出を採用した。これに
より層間比率は全数集計の真の値に一致する。NDL はサーバー側の暗黙 result cap（LIMIT 要求値に
よらず1応答最大1000行）を持つため、各層の抽出はページング（OFFSET を進めながら要求件数まで
取得）で行った。

### 3.3 グラフ変換

取得した1-hopトリプルから、`edge_type` 付き単一の `torch_geometric.data.Data` を構築する。

- **ノード**: 対象クラスのインスタンス + それらと1-hopで繋がる周辺エンティティ。
- **エッジ**: object property（値がuri型）をリレーションタイプとして保持。各リレーションに
  対して逆方向リレーションを明示的に追加する（双方向メッセージパッシング、R-GCN標準前処理）。
- **ラベルリーク防止**: ラベル定義に用いた `label.property` と `rdf:type` はエッジ・特徴量の
  自動発見から無条件除外する。これは R-GCN 原論文が AIFB/MUTAG 等で「ラベル定義に使った関係を
  グラフから除去する」のと同じ標準的前処理である。
- **ノード特徴量**: 構造特徴（次数 + リレーションタイプ別隣接数）+ 浅いテキスト特徴。テキストは
  `text_properties`（NDL: `rdfs:label`、DBLP: `dblp:title`）を文字n-gram (char_wb, 2–4) の
  hashing vectorizer で128次元にベクトル化する。文字n-gramを用いることで、分かち書きの無い
  日本語も英語と同じコードのまま扱える（言語ごとのif/elseを持たない）。語彙はエンドポイント
  ごとに独立に学習する。

### 3.4 モデルと評価

R-GCN（RGCNConv 2層 + 線形ヘッド1層）を、多数派クラスベースラインおよび特徴量のみベースライン
（グラフ構造を使わないロジスティック回帰）と比較する。特徴量のみベースラインは、R-GCN の性能が
グラフ構造由来かノード特徴由来かを切り分けるために不可欠である。評価指標は macro-F1・
balanced accuracy・混同行列、コスト（学習/推論時間・ピークメモリ）、および本研究の主眼である
**クラスセントロイド間コサイン類似度**（R-GCN 埋め込みからラベルごとの平均ベクトルを求め、
全クラスペアのコサイン類似度を算出。値が低いほどクラスがよく分離）である。データ分割は
train 0.7 / val 0.15 / test 0.15 の層化分割、乱数シードは42で固定した。

## 4. 結果と考察

数値は `scripts/run_training_eval.py`（RTX 5070 Ti / CUDA、乱数シード42）の出力
（`docs/results_*.md`・`docs/class_separation_summary.md`・`docs/class_separation_*.png`）から
転記した。グラフ規模は NDL: 対象15,000ノード（1-hop隣接含め総42,545ノード・55,112エッジ・
リレーション4種）、DBLP: 対象15,000ノード（総47,149ノード・100,000エッジ〈上限で間引き〉・
リレーション20種）である。

### 4.1 分類精度

| エンドポイント | R-GCN macro-F1 | 多数派 macro-F1 | 特徴量のみ macro-F1 |
|---|---|---|---|
| NDL | 0.7382 | 0.1655 | **0.7881** |
| DBLP | 0.5231 | 0.1351 | **0.5821** |

（balanced accuracy も同順で NDL: R-GCN 0.749 / 多数派 0.200 / 特徴量のみ 0.774、
DBLP: R-GCN 0.521 / 多数派 0.200 / 特徴量のみ 0.555。）

**最も重要かつ正直に報告すべき所見: 両エンドポイントとも、グラフ構造を一切使わない
「特徴量のみベースライン（ロジスティック回帰）」が R-GCN を上回った**（NDL 0.788 > 0.738、
DBLP 0.582 > 0.523）。これは失敗ではなく、まさにこのベースラインを無条件で置いた理由
（本課題の設計方針、決定事項#8）が機能した結果である。両ベースラインを併置したことで、
以下を明確に切り分けられた:

- 多数派ベースライン（NDL 0.166 / DBLP 0.135）に対しては R-GCN・特徴量のみとも大きく上回る。
  つまり両データセットとも「分類可能な信号」は確かに存在する。
- しかしその信号は**ノード自身の特徴（ここではラベル/タイトル文字列の文字n-gram）にほぼ
  尽きており、リレーション構造を足しても向上しない**。むしろ R-GCN では近傍集約が
  局所的な文字列信号をわずかに希釈し、精度を下げている。

この所見はデータの性質と整合的である。NDL では典拠概念どうしがほとんど相互リンクを持たず
（大半が VIAF・entity といった外部ノードへ次数1で接続し、15,000件中1,123件は完全に孤立）、
「典拠の種別（人名か地名か団体か）」は名前の表層形にほぼ現れる。DBLP でも「書誌タイプ
（Article か Inproceedings か）」はタイトルや掲載情報の文字列パターンに強く現れ、著者リンク
構造から追加で得られる情報は小さい。**「この2つの書誌ノード分類タスクでは、識別信号はノード
自身のテキストに宿り、リンク構造には乏しい」**というのが、比較分析から得られた具体的な知見で
ある。R-GCN を勝たせるためのモデル調整（層数・ドロップアウト等の探索）は、この所見を歪める
ため意図的に行っていない。

なお混同行列（`docs/results_*.md`）が示す通り、両データセットとも最小クラス（NDL の
uniformTitles〈test 8件〉、DBLP の Phdthesis・Incollection）はほぼ全て多数側クラスへ
誤分類される。これは top_k=5 が捉える裾クラスの極端な不均衡（決定事項#20 の全数分布）を
そのまま反映しており、macro-F1 が多数派ベースラインと大きく差がつく一方で完全分類には
至らない理由である。

### 4.2 クラス分離度の横断比較（本研究の主眼）

| エンドポイント | クラス数 | クラス間平均コサイン類似度 | R-GCN macro-F1 |
|---|---|---|---|
| NDL | 5 | **0.3837** | 0.7382 |
| DBLP | 5 | 0.4566 | 0.5231 |

クラス間平均コサイン類似度は、値が低いほどクラスのセントロイドが埋め込み空間上で離れている
＝分離しやすいことを意味する。**NDL（0.384）の方が DBLP（0.457）よりクラス分離度が高く、
これは NDL の方が R-GCN macro-F1 が高い（0.738 > 0.523）ことと整合する。** すなわち、埋め込み
空間上のクラスセントロイド分離という内在的指標が、実際の分類しやすさという外在的指標と同じ
向きを指しており、本指標がデータセット横断の「クラスの分離しやすさ」の要約として機能して
いることを支持する。

ただしこの2値の単純な大小比較は慎重に扱う必要がある。両データセットはクラス数こそ同じ5だが、
不均衡度・言語・スキーマ・リレーション数（NDL 4種 vs DBLP 20種）が大きく異なる。したがって
「NDL の方が普遍的に分類しやすい」と一般化するのではなく、「このパイプライン・この設定の下で、
NDL の典拠種別クラスは DBLP の書誌タイプクラスより埋め込み空間上でよく分離した」という限定
された主張にとどめる。各クラスペアの詳細は `docs/class_separation_*.png` のヒートマップを参照。

### 4.3 コスト

学習・推論はいずれも GPU 上で軽量である（R-GCN 学習: NDL 1.8s / DBLP 3.6s、推論: 20ms 前後、
ピーク GPU メモリ: NDL 250MB / DBLP 948MB）。**支配的なコストは学習ではなく SPARQL 抽出時間**で
あり、NDL 約58秒に対し DBLP は約1,036秒（約17分）を要した。これは DBLP が1書誌あたりの述語数が
多く（約25述語）、かつ 1-hop プロパティ取得を GET のURL長上限に収めるため60件/バッチに分割した
ことで往復回数が増えたためである（決定事項#23）。グラフ構築自体は両者とも1秒前後で、抽出時間が
規模に対して支配的というスケーリング傾向が確認できる。抽出結果は `data/raw/` にキャッシュされる
ため、再実行時のコストはゼロに近い。

## 5. 限界と今後の課題（隠さない）

1. **2エンドポイントへの縮小**: ICCU SBN・Cervantes Virtual は、全数集計で判明した追加の
   データ品質問題（ICCU SBN の異表記エイリアスTopic URI・極端な低被覆、Cervantes Virtual の
   空文字列最頻値・低被覆）への対応が未了のまま提出期限に達したため見送った。汎用パイプライン
   自体はこれら2件にも設定追加のみで適用可能な構造だが、「意味のあるラベルをどう定義するか」
   というデータ固有の判断が残っている。当初「異種スキーマ4件の横断比較」を掲げた本研究にとって、
   比較対象が2件に減ったことは主眼（横断比較）の説得力を弱める最大の限界である。
2. **変換時間スケーリング曲線を測っていない**: 課題仕様（`build-lod-project.md` Phase 6）は
   `max_nodes` を振って変換時間の増加傾向を `docs/conversion_time_scaling.png` に出すことを
   求めていたが、時間制約により実施できなかった。代わりに各エンドポイントの固定規模
   （15,000ノード）での SPARQL抽出時間とグラフ構築時間の内訳のみを記録した（4.3節）。抽出時間が
   支配的という定性的傾向は2点からでも読み取れるが、ノード数に対する定量的なスケーリング測定は
   今後の課題である。
3. **グラフ構造が精度に寄与しなかった**: 4.1節の通り、両データセットで特徴量のみベースラインが
   R-GCN を上回った。これは「書誌ノードのタイプ分類」というタスク選択に対して、リレーション構造
   が持つ追加情報が乏しかったことを意味する。R-GCN の利点を引き出すには、ノード自身のテキストでは
   解けない別のラベル（例: 研究分野・主題カテゴリなど、共著・引用ネットワークの構造に依存する
   ラベル）を分類対象に選ぶべきだった可能性がある。ラベル選定を全数集計の健全性のみで決め、
   「グラフ構造で解ける課題か」という観点を欠いていたことは設計上の反省点である。
4. **ラベル空間の非共有**: 各エンドポイントは独立グラフであり、クラス分離度という要約統計量
   自体を比較する（埋め込みベクトルそのものは比較しない）。
5. **層内サンプリング順序バイアス**: 層化抽出は層間比率のみ真値に一致させ、層内順序は
   サーバー既定順に依存する（真の一様ランダムではない）。
6. **未評価の頑健性軸**: グラフ構造ロバスト性・スケールアウト性・人為的クラス不均衡下の
   頑健性は時間制約により評価していない。
